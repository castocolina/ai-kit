# E7 Loop ‚Äî PLAN ledger

> **This file is the loop's memory.** Each iteration: read this top-to-bottom, find the first
> unchecked `- [ ]` task, do ONLY that task, verify it with a fresh subagent, then check its box
> and commit. Never skip ahead. Never check a box without PASS evidence. State lives here + in
> git ‚Äî not in your context (you may have compacted since the last iteration).
>
> **Design source of truth:** `docs/superpowers/specs/2026-06-20-e7-execution-design.md`.
> **Driver prompt + stop condition:** `docs/superpowers/LOOP-GUIDE-e7.md`.
> **Branch:** `feat/e7-loop` (off `main`; FR-7.1 already cherry-picked as `97f8cee`).
> **Tests:** `python3 -m unittest` (NOT pytest). `make test` runs the full suite. `make lint`.
> `python3 tools/status-line.py --doctor` self-check.

## Conventions per task
- **TDD**: write/extend the failing test FIRST (red), implement to green, then refactor.
- **Verify**: dispatch a FRESH general-purpose subagent (clean context) with the task's
  `verify:` command; it returns PASS/FAIL **with command output**. Box is checked ONLY on PASS.
- **UI tasks** (marked üé®): additionally dispatch `ui-ux-designer` at plan + exec + e2e.
- **Commit**: one atomic commit per task on PASS; message `feat(e7)/test(e7)/refactor(e7): ‚Ä¶`.
- After all tasks of an FR pass: run the **FR gate** (see ¬ßFR-GATES) before the next FR.
